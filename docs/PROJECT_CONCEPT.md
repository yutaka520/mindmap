# プロジェクト概念 — Mind Map

## 1. プロダクトの目的

**Mind Map** は、思考・企画・タスク・調査内容を、ローカル環境で素早く構造化するためのマインドマップアプリです。

最初の価値は「ブラウザを開いたらすぐ書けること」です。ログイン、クラウド同期、共同編集などは初期版に含めず、1人で考えを整理する道具として操作の速さと保存の確実さを優先します。

## 2. 想定ユーザーと利用場面

- 企画の発散と論点整理をしたい個人
- 開発やゲーム設計の構造を図で検討したい人
- 会議・調査・学習のメモを関係性ごとに残したい人
- ネットワークがなくても、ローカルで情報を扱いたい人

## 3. コア体験

1. 中心テーマを書き、ノードを追加する。
2. ノードを好きな場所に置き、内容を編集する。
3. ノードの上・右・下・左の接続点を選び、意図した方向で関係を結ぶ。
4. ブラウザに自動保存され、必要な時は JSON として書き出す。

「どの面からつなぐか」は、単なる見た目ではなく、思考の流れを表す操作です。たとえば、右を「展開」、下を「詳細」、左を「前提」のように自分で使い分けられます。

## 4. 初期版の機能範囲

- ノードの作成、選択、移動、名称編集、削除
- 四方向（上・右・下・左）を指定したノード接続
- ベジェ曲線によるエッジ描画と矢印表示
- キャンバスのパン・ズーム
- Local Storage への自動保存
- JSON の書き出し／読み込み
- ノード色の変更

## 5. 非目標（初期版では扱わないもの）

- アカウント・サーバー・クラウド同期
- 複数人の同時編集
- リッチテキスト、画像、添付ファイル
- 自動レイアウト
- モバイル向けの完全最適化

これらは基盤が安定してから、利用価値が高い順に追加します。

## 6. 設計原則

- **Local-first**: 編集内容はまず端末内に保存する。
- **関係を明示する**: エッジはどの面からどの面へ接続するかをデータとして保持する。
- **データを持ち出せる**: 独自DBに閉じず、JSONで復元可能にする。
- **操作を阻害しない**: ノード移動、接続、編集を少ない操作で完結させる。
- **型で壊れにくくする**: ノード、エッジ、接続面、ビューポートを TypeScript の型として定義する。

## 7. データモデル

```ts
interface MindMap {
  viewport: { x: number; y: number; zoom: number };
  nodes: MindMapNode[];
  edges: MindMapEdge[];
}

interface MindMapNode {
  id: string;
  text: string;
  x: number;
  y: number;
  color: 'blue' | 'amber' | 'violet' | 'root';
}

interface MindMapEdge {
  id: string;
  from: string;
  fromSide: 'top' | 'right' | 'bottom' | 'left';
  to: string;
  toSide: 'top' | 'right' | 'bottom' | 'left';
}
```

接続面を `fromSide` / `toSide` として保存するため、ノードを移動した後も、エッジは元の意図した面に追従します。

## 8. 技術方針

- Next.js App Router
- React 19
- TypeScript strict mode
- 依存ライブラリなしのSVGエッジ描画
- Client Component での Local Storage 利用

外部のキャンバスライブラリを初期段階で導入しないことで、接続仕様とデータ形式をアプリ要件に合わせて自由に進化させられます。

## 9. 次の拡張候補

優先度順の案です。

1. エッジ選択・削除・ラベル付与
2. ノードの複製、折りたたみ、検索
3. フォルダ／複数マップ管理
4. 自動レイアウト
5. Markdownエクスポート、PNG/PDFエクスポート
6. IndexedDB への保存と履歴（Undo / Redo）
7. 任意の同期機能
